## 🚨 세션 2: 미해결 버그 재현 결과

4개의 핵심 버그가 모두 재현되었습니다. (수정 필요)

### 1. TC-F001: 운동 삭제 기능 (BUG-029) 🟠 High
- **증상:** 운동 카드 내에서 모든 세트를 삭제해도 운동 카드가 사라지지 않음. 오히려 "최소 1개의 세트가 필요합니다"라는 경고가 뜨며 삭제 불가능. 운동 카드를 직접 삭제하는 버튼도 부재함.
- **결과:** ❌ **FAIL (버그 재현됨)**
- **조치:** 운동 카드 헤더에 '운동 삭제' 버튼을 추가하거나, 마지막 세트 삭제 시 운동 전체를 삭제할지 묻는 로직이 필요함.

### 2. TC-F002: 세트 추가 시 스크롤 (BUG-033) 🟡 Medium
- **증상:** 세트를 계속 추가할 때, 새로 추가된 세트가 화면 밖으로 밀려나지만 자동 스크롤이 되지 않아 사용자가 수동으로 스크롤해야 함.
- **결과:** ❌ **FAIL (버그 재현됨)**
- **조치:** `addSet` 동작 시 `ScrollController`를 사용하여 리스트 최하단으로 부드럽게 스크롤하는 기능 추가 필요.

### 3. TC-F003: 운동 선택 페이지 상세정보 (BUG-031) 🟡 Medium
- **증상:** 운동 선택 목록에서 운동명을 탭하면 상세 정보(`ExerciseDetailPage`)로 이동하지 않고, 단순히 운동이 선택(체크)됨.
- **결과:** ❌ **FAIL (버그 재현됨)**
- **조치:** 탭 동작을 분리해야 함. (예: 아이콘 탭은 선택, 텍스트 탭은 상세 정보 보기)

### 4. TC-F004: 최근 기록 버튼 (BUG-030) 🟡 Medium
- **증상:** 운동 카드 UI에서 "최근 기록" 버튼을 찾을 수 없음.
- **결과:** ❌ **FAIL (버그 재현됨 - 버튼 실종)**
- **조치:** 디자인 개편 과정에서 누락된 것으로 보임. `ExerciseCard` 헤더나 푸터에 버튼 복구 필요.
