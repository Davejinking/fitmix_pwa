# 🚀 1인 개발자 하이브리드 AI 워크플로우 (Kiro + Jules)

## 1. 핵심 철학: "병렬 처리(Parallel Processing)"

혼자 개발할 때 가장 큰 병목은 코딩과 테스트를 순차적으로 할 때 발생합니다. 이를 AI에게 분산시켜 해결합니다.

* **⚡ Kiro (Local AI):** **"행동대장"** (기능 구현, UI 수정, 실시간 디버깅)
* **🛡️ Jules (Cloud AI):** **"품질관리팀"** (테스트 작성, 환경 검증, 문서화)

---

## 2. 사전 준비 (환경 설정)

쥴스(Jules)가 세션 초기화 후에도 즉시 일할 수 있도록 설정 파일을 준비합니다.

### A. 폴더 구조 (테스트 전략)

```text
test/
├── bugs/                   # 🚑 버그 픽스 검증용 (최우선)
│   ├── bug_001_login_fail_test.dart
│   └── bug_017_timer_crash_test.dart
├── features/               # 📱 주요 기능 시나리오 (P0)
│   └── exercise_flow_test.dart
└── setup_tests.sh          # ⚙️ 쥴스 자동 세팅 스크립트

```

### B. 자동화 스크립트 (`setup_tests.sh`)

프로젝트 루트에 생성합니다.

```bash
#!/bin/bash
# setup_tests.sh
echo "🚀 [Start] Flutter 테스트 환경 설정 중..."

# 1. 패키지 설치
flutter pub get

# 2. 코드 생성 (Freezed/JSON Serializable 사용하는 경우 주석 해제)
# dart run build_runner build --delete-conflicting-outputs

echo "✅ [Complete] 테스트 준비 완료! 'flutter test'를 실행하세요."

```

## 3. 실전 개발 루프 (The Loop)

### 🔄 Step 1: 기능 개발 & 핫픽스 (With Kiro)

**도구:** 로컬 에디터 (Kiro/Cursor) + 시뮬레이터
**목표:** 빠르게 돌아가는 코드 만들기

1. `flutter run`으로 앱 실행.
2. Kiro에게 명령하여 기능 구현 또는 버그 수정.
> "운동 상세 페이지에서 텍스트 오버플로우 나는 거 해결해줘."


3. 눈으로 UI 확인 후 기능이 작동하면 `git push`.
* *(이때 테스트 코드는 작성하지 않음 - 시간 절약)*



### 🔄 Step 2: 품질 검증 위임 (With Jules)

**도구:** 웹 브라우저 (Jules)
**목표:** 내가 짠 코드를 남(AI)이 검증하게 하기

1. Jules 세션 시작.
2. **초기 세팅 명령:**
> "Git에서 최신 코드 당겨오고, `setup_tests.sh` 실행해서 환경 세팅해줘."


3. **테스트 작성 명령:**
> "방금 업데이트된 `exercise_detail_page.dart` 확인해. 이 기능이 정상 작동하는지 검증하는 `test/bugs/bug_xxx_test.dart` 파일을 작성하고, 실제로 테스트를 돌려서 통과하는지 보여줘."



### 🔄 Step 3: 개발자는 다음 작업 진행 (With Kiro)

Jules가 테스트 코드를 짜고 돌리는 시간(5~10분) 동안 멍하니 있지 않습니다.

* 개발자는 로컬에서 **다음 버그(Bug-003)**를 수정하거나 **새 기능**을 개발합니다.

### 🔄 Step 4: 합치기 (Merge)

1. Jules가 "테스트 통과했습니다"라고 하면 코드를 Push 또는 PR 하라고 지시.
2. 로컬에서 `git pull` 수행.
3. **결과:** 내 컴퓨터엔 **[수정된 기능 + 검증된 테스트 파일]**이 모두 들어오게 됨.

## 5. 1인 개발자를 위한 조언

1. **완벽주의를 버리세요:** 모든 기능에 테스트 코드를 짤 수 없습니다. **"한 번 터졌던 버그(Regression)"**만 테스트 코드로 남겨도 충분합니다.
2. **Jules는 '신입 사원'입니다:** 가끔 엉뚱한 코드를 짤 수 있습니다. 하지만 테스트 코드는 메인 로직에 영향을 안 주므로, 신입에게 맡기기 가장 좋은 업무입니다.
3. **컨텍스트 스위칭 최소화:** 창작(Coding)과 검증(Testing)을 뇌에서 분리하세요. 창작은 님이 하고, 검증은 기계가 합니다.
