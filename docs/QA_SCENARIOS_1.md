## 2. 내일 바로 할 일 리스트 (Top 10)

위 시나리오 중 우선순위가 높고 즉시 검증이 필요한 항목들입니다.

1.  **[B-01] 저장 중 화면 이탈 방어 테스트 (P0)**: `await repo.put()` 도중 뒤로가기 시 앱이 죽는지 확인.
2.  **[C-01] 완료 버튼 따닥(Double Tap) 방지 (P0)**: 중복 저장으로 데이터가 2배 뻥튀기되는지 확인.
3.  **[D-01] 운동 수정 후 데이터 무결성 확인 (P0)**: 캘린더의 볼륨 합계가 수정된 값으로 즉시 바뀌는지.
4.  **[E-01] Null 데이터 조회 방어 (P0)**: DB 초기화 직후 캘린더 진입 시 에러 로그 없는지.
5.  **[G-03] 실수로 인한 운동 종료 방지**: 운동 중 뒤로가기 눌렀을 때 다이얼로그가 뜨는지(PopScope).
6.  **[G-01] 예외 처리 UX**: 저장 실패 시 유저가 인지할 수 있는 피드백(SnackBar)이 뜨는지.
7.  **[A-03] 빈 세션 저장 방지**: 운동 없이 저장되어 쓰레기 데이터가 쌓이는지 확인.
8.  **[F-02] 순서 변경 저장**: 운동 순서를 바꿨는데 재진입하면 원래대로 돌아가는지(State만 바뀌고 DB 저장 안되는 버그 흔함).
9.  **[B-02] 백그라운드 타이머**: 유튜브 보다가 돌아왔을 때 휴식 타이머가 정상적으로 흘렀는지.
10. **[A-02] 음수 값 입력 방어**: -100kg 입력 시 통계 그래프가 망가지는지 확인.
